Note that this is only a brief guide to show the use of the user authentication, for more detail about the build, configuration or anything else, please refer to the documentation on the GitHub project.

What is it?

AngularJs-Login-Authentication is an example of how to control and authenticate users using a REST API with AngularJS.

How does it work?

Upon signing up or logging in, the REST API responds with a session token which is then stored in the browser's cookies folder which can then be used to authenticate whether or not a user is logged in.

Often you will want some pages to only be accessible to logged in users, you can specify these pages in the resolve of the config block, using the now stored session token to authenticate the user.

Inside the login and signup controllers, on a successful call, it will initiate a $location.path change to the user's profile.

I left the profile tab on the menu to show that when trying to access the profile page without being logged in you will be unable to access it and redirected to the home page.
